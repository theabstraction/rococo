Sexy Scripting Language - Raw S-Expression

It is possible to store arbitrary data within a Sexy file, and use it as a data repository, something similar to an XML file. This is done with the apostrophe '.

Any compound expression with first character apostrophe is ignored by the Sexy compiler, as in the example below: 


	(' this is raw data)
In the MPlatform environment, Sexy supports a very limited pre-processor set. In fact, just one pre-processor command. This has the format (' #include <file1> <file2> ... <fileN>). This command is only parsed for the main script file and adds all files in the include statement to the source bundle for compilation. The same technique can be used to add any meta data to a file for use by third-party parsers.

In addition it is possible to define an expression variable as exampled by the following syntax: 


	(IExpression s = ' (this is raw data))
As we can see s is a variable of type Sys.Type.IExpression which is initialized to an expression (this is raw data). The various methods of IExpression can be used to pull out the tokens. This allows you to embed arbitrary data in a program, create a reference to it and then process that data, which saves having to load a separate file and write a parser for it.
